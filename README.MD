# Ptech Auto

Плагин для Figma, который собирает листовки и каталоги из `JSON`, подтягивает изображения товаров, сортирует карточки по `CSV`, а также обновляет скидки, цены и баннер.

## Что умеет

- Генерировать каталог из `JSON`, дизайнер делает выгрузку по стандартной инструкции из 1с и ставит галочку `Экспортировать в JSON`
- Подтягивать фото товаров через прокси (происходит автоматически)
- Сортировать карточки по порядку из `CSV/TXT` (автоматика)
- Сразу при генерации подставлять скидки и условия из того же `CSV` (автоматически)
- Обновлять на странице скидки, цены и текст баннера (автоматически)
- Добавлять футер на последнюю страницу (автоматически)
- Разбивать длинные карточки, если они не влезают в колонку (автоматически)

## Установка из релиза

Сборка уже выложена в [GitHub Releases](https://github.com/Polytech-Instrument/Figma-plugins/releases/tag/stable):

1. [Скачать архив релиза](https://github.com/Polytech-Instrument/Figma-plugins/releases/tag/stable) `любой из трех архивов, но можно просто самый первый с указанием версии`
2. Распаковать его в удобную папку
3. В Figma открыть `Plugins` -> `Development` -> `Import plugin from manifest...`
4. Выбрать файл `manifest.json` `<ПУТЬ_ДО_АРХИВА>/dist/manifest.json`

После этого плагин появится в локальных development plugins.

## Как пользоваться

### 1. Генерация каталога
![alt text](https://github.com/Polytech-Instrument/Figma-plugins/blob/main/readme_media/screen1.jpg)

Нужно загрузить:

- `JSON` с товарами
- `CSV/TXT` с порядком, скидками и условиями

Опционально:

- включить `Блок подарков` (увеличивает сдвиг в зависимости от высоты блока подарков, настраивается)
- включить `Компактную верстку` (нарушает порядок верски в угоду компактности)
- использовать `Тестовую генерацию` (для проверки вывода товаров, генерирует первые 10 позиций из файла)

Что делает генерация:

1. Читает `JSON`
2. Читает `CSV/TXT`
3. Определяет порядок карточек по колонке `Код` или по первой числовой колонке
4. Берет скидку из колонки `Скидка без условий`
5. Берет условия из колонки `Условия`
6. Подтягивает изображения товаров
7. Собирает страницы в Figma

### 2. Обновление информации
![alt text](https://github.com/Polytech-Instrument/Figma-plugins/blob/main/readme_media/screen2.jpg)

Нужно загрузить `CSV`.

Что обновляется:

- скидка в карточке
- цена, если в таблице есть колонка цены
- `Условия` из колонки `Условия`
- данные баннера:
  - максимальная скидка
  - месяц следующей акции
  - диапазон дат акции

Обновление идет по артикулу.

## Формат JSON

Ожидается массив групп товаров.

Пример:

```json
[
  {
    "id": "1",
    "title": "АДАПТЕРЫ ДЛЯ ПАТРОНОВ",
    "brand": "2",
    "description": "Адаптер SDS+ на резьбу патрона",
    "variants": [
      {
        "sku": "4978001",
        "specs": "на резьбу 1/2\"",
        "min": "1",
        "qty": "300",
        "price": "75.75"
      }
    ]
  }
]
```

## Формат CSV / TXT

Поддерживается таблица, где:

- служебные строки в начале можно пропускать
- порядок определяется по колонке `Код`
- скидка берется из `Скидка без условий`
- условия берутся из `Условия`

Пример заголовков:

```text
Код | Номенклатура | Дата выгрузки | Скидка без условий | Условия
```

Если заголовка `Код` нет, плагин пытается начать чтение с первой строки, где в первом столбце идет числовой артикул.

***наша стандартная таблица идеально подходит структурно***

## Настройки

В плагине есть настройки макета. Через них можно менять:

- размеры страницы
- отступы
- ширину колонок
- расстояния между элементами
- высоту футера
- авторазбивку длинных карточек

## Особенности верстки

- Без блока подарков первая страница стартует со сдвигом `1/3` от высоты `A4`
- С блоком подарков первая страница стартует со сдвигом `2/3` от высоты `A4`
- Если карточка слишком длинная, плагин может разбивать ее на несколько частей
- Для очень длинных карточек используется разбиение с учетом количества строк и доступного места в колонке

## Что важно для шаблона Figma

Плагин опирается на имена слоев в компонентах.

Основные имена:

- `#Title`
- `#Description`
- `#Image`
- `productDiscount`
- `#DiscountInfo`
- `saleStar`
- `#ListContainer`
- `#SKU`
- `#Specs`
- `#Min`
- `#Qty`
- `#Price`
- `itemSale`
- `priceProductDiscount`

Также используются:

- компонент карточки товара
- компонент строки товара
- компонент баннера
- компонент футера

Если переименовать эти слои в шаблоне, логика перестанет находить нужные элементы.

## Известные ограничения

- Для загрузки изображений нужен доступ к прокси (прокси развернуто на сервере, иногда при старте плагина может быть задержка до 90 секунд между тем как нажмете генерация и до того как прогрессбар покажет что начал сборку, это нормально, свыше 120 секунд - не нормально, стоит запустить генерацию заново или связаться с разработчиком)
- Качество генерации зависит от структуры входного `JSON`
- Разбиение длинных карточек зависит от структуры карточки и высоты контента в шаблоне

## Полезно для отладки

Если что-то работает не так, в первую очередь проверить:

- правильный ли `manifest` подключен в Figma
- совпадают ли имена слоев в шаблоне
- есть ли в `CSV` корректная колонка `Код`
- есть ли в `JSON` корректные `sku`
